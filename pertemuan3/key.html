Key

Apa itu Key dalam ERD?

Key dalam Entity Relationship Diagram (ERD) adalah atribut yang dipilih untuk mengidentifikasi entitas secara unik. Keunikan atribut key memungkinkan entitas tersebut dibedakan dari entitas lain dalam basis data.

Fungsi Key

Key dalam ERD berfungsi untuk memberikan identifikasi pada setiap baris data dalam sebuah tabel dan sebagai penghubung antara satu tabel dan juga dengan tabel lainnya. Key juga sangat penting sebagai tanda pengenal dan penghubung antar tabel dalam suatu relational database. Key dapat dipahami sebagai ‘tanda pengenal’ untuk mengidentifikasi sebuah baris di dalam tabel. Key juga berperan penting sebagai penghubung antara satu tabel dan juga dengan tabel lainnya. Key dalam database juga berfungsi untuk menjalankan fungsi sebagai ‘tanda pengenal’ dan sebagai penghubung antar tabel.

Primary Key

Primary key adalah suatu nilai yang ada didalam suatu basis data yang dimana digunakan untuk mengidentifikasi suatu baris yang ada di dalam tabel. Nilai yang ada didalam primary key adalah unik. Sedangkan secara sederhananya primary key dapat juga diartikan sebagai kolom yang berisi nilai unik, yang dimana memiliki fungsi sebagai identitas yang untuk membedakan setiap record yang ada didalam suatu tabel.

Primary key juga dapat diartikan sebagai atribut atau kumpulan atribut yang dapat mengidentifikasi setiap baris data dalam sebuah relasi secara unik. Primary key juga dikenal sebagai kunci utama karena merupakan kunci yang paling penting dalam sebuah tabel. Primary key berfungsi sebagai tanda pengenal dan penghubung antar tabel dalam suatu relational database.

karakteristik dari primary key adalah:

  • Unik: Setiap nilai dalam primary key harus unik, tidak boleh ada duplikasi.
  • Tidak Null: Primary key tidak boleh memiliki nilai null.

Primary key juga dapat terdiri dari beberapa kolom, yang dikenal sebagai composite key.

Contoh Penerapan Primary Key

  1. Tabel Mahasiswa

    2. Dalam tabel mahasiswa, NIM (Nomor Induk Mahasiswa) dapat dijadikan sebagai primary key karena setiap mahasiswa memiliki NIM yang unik.

  2. Tabel Anggota

    3. Dalam tabel anggota, id_anggota dapat dijadikan sebagai primary key karena setiap anggota memiliki ID yang unik.

  3. Tabel Karyawan

    4. Dalam tabel karyawan, id_karyawan dapat dijadikan sebagai primary key karena setiap karyawan memiliki ID yang unik.

  4. Tabel Pelanggan

    5. Dalam tabel pelanggan, id_pelanggan dapat dijadikan sebagai primary key karena setiap pelanggan memiliki ID yang unik.

  5. Tabel Pasien

    6. Dalam tabel Pasien, no_pasien dapat dijadikan sebagai primary key karena setiap pasien memiliki ID yang unik.

Foreign Key

Foreign Key adalah atribut atau kumpulan atribut yang digunakan untuk menghubungkan antara tabel master dengan tabel turunannya. Foreign Key digunakan untuk menciptakan hubungan antara dua tabel dalam database relasional. Foreign Key juga dikenal sebagai kunci tamu karena merupakan kunci yang terdapat pada tabel kedua atau child. Foreign Key merujuk pada suatu kolom yang unik untuk menghubungkan antar tabel. Foreign Key biasanya merujuk pada Primary Key di tabel lain, sehingga dapat memastikan integritas referensial antara tabel-tabel tersebut.

Fungsi Foreign Key

Foreign Key berfungsi sebagai penghubung antar tabel dalam suatu relational database. Foreign Key juga dapat digunakan untuk mengatur hubungan antar tabel, seperti one-to-one, one-to-many, dan many-to-many.

karakteristik dari foreign key adalah:

  • Referensial: Foreign key merujuk pada primary key di tabel lain.
  • Nilai Null: Foreign key dapat memiliki nilai null jika tidak ada hubungan yang sesuai di tabel referensi.

Contoh Penerapan Foreign Key

  1. Tabel Prodi dan Tabel Mahasiswa

    2. Dalam tabel mahasiswa, kode_prodi dapat dijadikan sebagai foreign key yang merujuk pada primary key kode_prodi di tabel prodi. Hal ini menunjukkan bahwa setiap mahasiswa terdaftar dalam sebuah program studi tertentu.

  2. Tabel Karyawan dan Tabel Perusahaan

    3. Dalam tabel karyawan, perusahaan_kode_perusahaan dapat dijadikan sebagai foreign key yang merujuk pada primary key kode_perusahaan di tabel perusahaan. Hal ini menunjukkan bahwa setiap karyawan bekerja di sebuah perusahaan tertentu.

  3. Tabel Pelanggan dan Tabel Transaksi

    4. Dalam tabel transaksi, id_pelanggan dapat dijadikan sebagai foreign key yang merujuk pada primary key id_pelanggan di tabel pelanggan. Hal ini menunjukkan bahwa setiap transaksi dilakukan oleh seorang pelanggan tertentu.

  4. Tabel Buku, Tabel Peminjaman dan Tabel Anggota

    5. Dalam tabel peminjaman, Buku_ISBN dapat dijadikan sebagai foreign key yang merujuk pada primary key ISBN di tabel buku, dan id_anggota dapat dijadikan sebagai foreign key yang merujuk pada primary key id_anggota di tabel anggota. Hal ini menunjukkan bahwa setiap peminjaman dilakukan oleh seorang anggota tertentu dan melibatkan sebuah buku tertentu.

  5. Tabel Pasien dan Tabel Dokter

    6. Dalam tabel pasien, id_dokter dapat dijadikan sebagai foreign key yang merujuk pada primary key id_dokter di tabel dokter. Hal ini menunjukkan bahwa setiap pasien ditangani oleh seorang dokter tertentu.

Candidate Key

Candidate Key adalah kumpulan atribut yang dapat membedakan setiap baris data dalam sebuah relasi secara unik. Candidate Key juga dikenal sebagai kunci calon karena merupakan kunci yang memenuhi syarat untuk menjadi Primary Key. Candidate Key berfungsi sebagai tanda pengenal dan penghubung antar tabel dalam suatu relational database. Candidate Key dapat terdiri dari satu atau lebih atribut yang dapat mengidentifikasi setiap baris data dalam sebuah tabel secara unik.

karakteristik dari candidate key adalah:

  • Unik: Setiap nilai dalam candidate key harus unik, tidak boleh ada duplikasi.
  • Tidak Null: Candidate key tidak boleh memiliki nilai null.

Contoh Penerapan Candidate Key

  1. Tabel Mahasiswa

    2. Dalam tabel mahasiswa, NIM (Nomor Induk Mahasiswa) dan email dapat dijadikan sebagai candidate key karena setiap mahasiswa memiliki NIM dan email yang unik.

  2. Tabel Karyawan

    3. Dalam tabel karyawan, id_karyawan dan email dapat dijadikan sebagai candidate key karena setiap karyawan memiliki ID dan email yang unik.

  3. Tabel Pelanggan

    4. Dalam tabel pelanggan, id_pelanggan dan email dapat dijadikan sebagai candidate key karena setiap pelanggan memiliki ID dan email yang unik.

  4. Tabel Pasien

    5. Dalam tabel pasien, no_pasien dan NIK dapat dijadikan sebagai candidate key karena setiap pasien memiliki ID dan NIK yang unik.

  5. Tabel Anggota

    6. Dalam tabel anggota, id_anggota dan no_ktp dapat dijadikan sebagai candidate key karena setiap anggota memiliki ID dan no_ktp yang unik.
